React Auth Kit is a lightweight, production-ready authentication library for React-based applications. It handles JWT token storage, automatic expiry checking, token refresh, and private route protection — so you can focus on building your app instead of reinventing auth plumbing.

## Why React Auth Kit?

Managing authentication tokens in React has always required manual effort: storing tokens, checking expiry, refreshing sessions, and redirecting unauthenticated users. React Auth Kit automates all of this with a clean, hook-based API.

## How it works

React Auth Kit uses a reactive store (powered by RxJS) to hold your authentication state — the access token, its expiry, any refresh token, and optional user data. The store is created once and passed to `AuthProvider`, which makes all hooks available throughout your component tree.

<Steps>
  <Step title="Create a store">
    Call `createAuthStore` to configure your token storage (cookie or localStorage) and optional refresh settings.
  </Step>
  <Step title="Wrap your app with AuthProvider">
    Pass the store to `AuthProvider` at the root of your application. All child components can now use React Auth Kit hooks.
  </Step>
  <Step title="Sign in with useSignIn">
    After authenticating with your backend, call `useSignIn` to store the JWT and user state.
  </Step>
  <Step title="Access auth state anywhere">
    Use `useIsAuthenticated`, `useAuthUser`, and `useAuthHeader` in any component to read the current auth state.
  </Step>
</Steps>

## Packages

React Auth Kit is a monorepo. Install the packages you need:

| Package | Description |
|---|---|
| `react-auth-kit` | Core library — store, provider, and all hooks |
| `@auth-kit/next` | Plugin for Next.js App Router private routes |
| `@auth-kit/react-router` | Plugin for React Router private routes |

<Tip>
  If you only use React without a specific router, the core `react-auth-kit` package is all you need.
</Tip>
